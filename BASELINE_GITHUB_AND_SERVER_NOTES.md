# MACT Baseline GitHub and Server Notes

This repository is a local MACT baseline copy prepared for reproducible comparison experiments against myAgent.

Original project:

Efficient Multi-Agent Collaboration with Tool Use for Online Planning in Complex Table Question Answering, NAACL 2025.

License:

The upstream code is distributed under AGPL-3.0. Keep `LICENSE.txt`, `3rd-party-licenses.txt`, the original README, and citation information when pushing this baseline to GitHub.

## What was changed locally

This local copy includes runner/backend compatibility changes so MACT can be executed under the same experimental conditions as myAgent:

| File | Purpose |
|---|---|
| `code/model_backends.py` | Adds DeepSeek/OpenAI-compatible chat-completions backend, provider parameters, retry/timeout configuration, and token tracking. |
| `code/tqa.py` | Adds backend selection, explicit output paths, append/overwrite control, and per-sample `api_metrics`. |
| `code/agents.py` | Routes MACT LLM calls through the injected API backend when configured. |
| `tests/` | Adds regression tests for API backend behavior and entry-point compatibility. |

These changes are only for running MACT with the same model interface, input format, and metric logging used in the myAgent experiments. They do not add myAgent's patent mechanisms such as risk-aware routing, table compression policy, deterministic shortcuts, post-verification, or selective collaboration budget control.

## What should be pushed to GitHub

Push this MACT baseline as a separate repository, not inside the myAgent repository.

Recommended repository name:

`mact-baseline-deepseek-runner`

Recommended visibility:

Private, unless you intentionally want to publish the modified AGPL baseline.

Do not push:

- API keys or `.env` files
- `outputs/`
- full datasets
- model checkpoints
- temporary logs

The `.gitignore` in this repository already excludes those paths.

## DeepSeek API run example

Set the API key only in the environment:

```powershell
$env:DEEPSEEK_API_KEY="YOUR_KEY_HERE"
```

Run WTQ with the adapted JSONL file generated by myAgent:

```powershell
python code/tqa.py `
  --task wtq `
  --dataset_path D:\AAAcode\code-code\agent+\myAgent-main\datasets_ready\blind_holdout_200_v1_2026-06-27\wtq.jsonl `
  --output_path outputs\blind200\wtq_mact.jsonl `
  --plan_model_name deepseek-v4-flash `
  --code_model_name deepseek-v4-flash `
  --model_provider deepseek `
  --api_base https://api.deepseek.com `
  --api_key_env DEEPSEEK_API_KEY `
  --thinking disabled `
  --temperature 0 `
  --max_tokens 2048 `
  --plan_sample 1 `
  --code_sample 1 `
  --max_step 3 `
  --max_actual_step 3
```

For TabFact, use `--task scitab` with the TabFact JSONL path. For CRT, use `--task crt`.

## Evaluation

Evaluate MACT output with myAgent's shared evaluator:

```powershell
python D:\AAAcode\code-code\agent+\myAgent-main\code\evaluate_results.py `
  outputs\blind200\wtq_mact.jsonl `
  --error_output outputs\blind200\wtq_mact_errors.jsonl
```

Use the same evaluator for myAgent and MACT outputs. This is the key fairness rule for thesis experiments.

## Server recommendation

For formal experiments, the cleanest protocol is:

1. Clone myAgent and this MACT baseline repo separately.
2. Record the MACT baseline repo commit hash.
3. Use the same adapted dataset JSONL files for both systems.
4. Use the same model endpoint and model name for both systems.
5. Save raw JSONL outputs, evaluator JSON summaries, error JSONL files, and run logs.
